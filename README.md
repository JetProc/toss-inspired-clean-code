# toss-inspired-clean-code

토스 프론트엔드 클린 코드 스타일에서 아이디어를 얻어, 직접 관련 문제를 만들어 리액트 컴포넌트를 더 읽기 좋고 유지보수하기 쉬운 구조로 리팩토링하는 연습 저장소입니다.

공식 자료를 그대로 옮기기보다, 비슷한 문제 상황을 스스로 구성한 뒤 `before`와 `after`를 비교하면서 컴포넌트 분리, 조건 분기 정리, 추상화 레벨 조절을 연습합니다.


## 구조

```text
src/
  day{n}/
    before.{tsx,txt}
    after.{tsx,txt}
    ...
```

`day{n}` 디렉터리는 학습 순서대로 추가됩니다.

- `before.*`: 읽기 어렵거나 책임이 섞인 예제
- `after.*`: 같은 동작이나 구조를 더 명확하게 정리한 풀이
- 그 외 파일: 해당 예제 풀이에 필요한 훅, 유틸, 타입 등

## 다루는 주제

### 🟢 [1부] 가독성 (Readability)

- [x] Day 1: 이름에 의도 담기 - 모호한 변수명 대신 구체적이고 맥락이 담긴 이름 짓기
- [x] Day 2: 부정 조건문 피하기 - 뇌를 꼬이게 하는 이중 부정을 지양하고 긍정 조건식으로 작성하기
- [x] Day 3: 조기 반환(Early Return)으로 중첩 줄이기 - 끝없는 `if/else` Depth를 `return`으로 방어하기
- [x] Day 4: 삼항 연산자 단순하게 유지하기 - 가독성을 해치는 중첩 삼항 연산자를 피하기
- [x] Day 5: 매직 넘버에 이름 붙이기 - 코드 속 의미 모를 숫자를 읽기 쉬운 상수로 캡슐화하기
- [x] Day 6: 복잡한 조건식 캡슐화하기 - 길어진 조건문을 의미를 담은 변수나 함수로 빼내기
- [x] Day 7: 객체 구조 분해 할당 활용하기 - 불필요하게 반복되는 객체 체이닝 깔끔하게 정리하기
- [x] Day 8: 함수는 하나의 일만 하도록 쪼개기 (단일 책임 원칙) - 수백 줄짜리 함수를 작고 읽기 쉽게 분리하기

### 🔵 [2부] 예측 가능성 (Predictability)

- [x] Day 9: 섀도잉(Shadowing) 방지하기 - 외부와 내부 스코프의 변수명을 다르게 지어 변수 가림 현상 막기
- [x] Day 10: 함수의 반환 타입 일치시키기 - 상황마다 다른 형태를 반환하지 않고, 통일된 반환 객체 구조 만들기
- [x] Day 11: 숨은 로직 드러내기 - 원본 데이터를 훼손하는 부수 효과(Side Effect)를 없애고 순수 함수 지향하기

### 🟡 [3부] 응집도 (Cohesion)

- [x] Day 12: 함께 수정되는 파일을 같은 디렉토리에 두기 - 도메인(Feature) 단위로 컴포넌트, 훅, API 묶어주기
- [x] Day 13: 매직 넘버 없애기 (설정 데이터 중심화) - 전역에 흩어진 비즈니스 정책 숫자를 하나의 상수 객체로 모으기
- [x] Day 14: 연관된 상태(State) 하나로 묶기 - 낱개로 찢어진 폼(Form) 상태들을 하나의 객체로 모으기
- [x] Day 15: 비즈니스 로직을 커스텀 훅으로 분리하기 - UI 컴포넌트에서 상태와 로직을 찢어내어 `use~` 훅으로 응집하기

### 🔴 [4부] 결합도 (Coupling) - 🏃‍♂️ 현재 진행 중!

- [x] Day 16: 도메인 결합도 낮추기 (객체 통째로 넘기지 않기) - 컴포넌트가 특정 도메인을 모르게 원시 타입 Props만 받도록 하기
- [ ] Day 17: 제어권을 부모에게 위임하기 (제어 역전) - 자식이 쥐고 있던 렌더링 제어권을 밖으로 꺼내 유연성 극대화하기
- [ ] Day 18: 합성 컴포넌트로 UI 조립 가능하게 만들기 - 복잡한 UI를 여러 개의 작은 조각으로 나누어 외부에서 조립하게 하기
- [ ] Day 19: 불필요한 데이터 통로 끊어내기 (Props Drilling 걷어내기) - 컴포넌트 사이를 관통하는 데이터를 깔끔하게 주입하기
- [ ] Day 20: 프론트엔드 아키텍처 최종 리뷰 및 리팩토링 - 4가지 핵심 원칙을 모두 적용해 최악의 레거시 코드를 개선하는 졸업 프로젝트

## 사용 방법

의존성을 설치한 뒤 타입 검사를 실행합니다.

```bash
npm install
npm run typecheck
```

새 예제를 추가할 때는 `src/day{n}` 디렉터리를 만들고, 가능하면 같은 동작이나 구조를 기준으로 `before`와 `after`를 함께 둡니다.
